MySQL对大小写不敏感

登陆数据库：mysql -uroot -p密码；
退出数据库服务器命令：exit；

1.show databases; 查看现有数据库
2.create database 名字; 新建数据

使用数据库（后来才能查询等）：use 数据库名字;
查询数据库中所有表格：show tables；
查询表格：select * from 表名;【可以用where加条件如：select * from student where 学号=19;】

create table 表名(
属性名 数据类型 [约束条件],
属性名 数据类型 [约束条件],
属性名 数据类型 [约束条件]
);
[]中括号种的内容是可选的
最后一个属性后面没有逗号

eg：
create table student(
学号 int,
名字 varchar(10),
年龄 int,
性别 varchar(5)
);

查看创建好的表的数据结构：describe 表名；

往数据表中添加数据记录：insert into 表名 values();

如：insert into teacher values(1,’2’,NULL,3,’1999-4-14’)

可以指定插入的字段

如：insert into teacher (名字) values (‘翁瀚’);

删除记录：delete from 表名 where 字段名=xxxx;

如：delete from student where 名字=’张美丽’;

改记录数据：update 表名 set 字段名=xxx where 字段名=xxx;

如：update student set 名字 = ‘翁瀚’ where 名字 = ‘张美丽’;

很多种数据类型

double类型使用
实例：double(10,2)//长度为10，小数点2

mysql建表约束：

1 主键约束 primary key

它能够唯一确定表中的一条记录，也就是我们通过给某个字段添加约束，就可以使得该字段不重复且不为空

如：
create table user
( id int(5) primary key,

  • name varchar(20)*
  • )*
  • 或者如：✔*
  • create table user_1*
  • (
  • id int,*
  • name varchar(20),*
  • password varchar(20),*
  • primary key(id)*
  • )*

2 联合主键 primary key()

只要两个或者多个主键值加起来不重复就好

如：
create table user_1
(

  • id int,*
  • name varchar(20),*
  • password varchar(20),*
  • primary key(id,name)*
  • )*

3 自增约束 auto_increment

能自动增长某一个添加了自增约束的字段。若添加记录时没有添加该字段，则该字段自动生成且为表中最后一个记录记录同字段+1；

create table use_2( id int primary key auto_increment,name varchar(20));

当只添加名字时候，insert into use_2 (name) values(‘zhangshan’);
数据库会按顺序自动给生成id。此时id=1
再添加 insert into use_2 (name) values(‘zhangwei’);
数据库会按顺序自动生成id。此时id=2

4 外键约束 foreign key,references

涉及两个表：父表和子表 或者叫 主表和副表
a.主表中没有的数据值；在副表中是不可以使用的
b.主表中的记录一旦被副表引用，是不能被删除的

主表：(被副表引用的字段要是主键）
create table class
(
id int primary key,
name varchar(20)
);

副表（记得是references）
create table students
(
id int,
name varchar(20),
class_no int,
foreign key(class_no) references class(id)
);

5 唯一约束

约束修饰字段的值不可以重复，如果给两个字段添加唯一约束，那么这两个字段加起来不能重复（类似于联合主键）

如：
create table user_5 *
*( *
*id int, *
*name varchar(20), *
*unique(id) *
*); *
*或者如：
create table user_5
(
id int unique,
name varchar(20)
);

6 非空约束 not null

修饰的字段不能为空

如：(只有一种方式）
create table user_3
(

  • id int,*
  • varchar(20) not null*
  • );*

7 默认约束 default

就是当我们插入字段值的时候，如果没有传值，就会使用默认值；但是传了值就不会使用默认值

如：(只有一种方式）

  • create table user_2*
  • (*
  • id int,*
  • name varchar(10),*
  • age int,*
  • birth date default ‘1999-4-14’*
  • );*

创建表后添加主键约束：

第一种方式(使用add)alter table 表名 add primary key（字段名）；✔
如：alter table student add primary key(名字);
第二种方式(使用modify)add table 表名 motify 字段名 类型名 primary key
如：
alter table student modify 学号 int(11) primary key;

创建表后删除主键约束：alter table 表名 drop primary key;✔
如：alter table student drop primary key;

创建表后添加唯一约束：

第一种方式(使用add) alter table student add unique(字段名);✔
如：alter table student add unique(学号);

第二种方式(使用modify)add table 表名 motify 字段名 类型名 unique;
如：
alter table student modify 学号 int(11) unique;

创建表后删除主键约束：alter table 表名 drop index 字段名;✔
如：alter table student drop index name;

改变外键值
将外键的值设置为0，此时就是失效状态，命令：set foreign_key_checks = 0，这时就能删除数据了
删除之后，外键再设置为有效：

删除列：alter table 表名 drop 列名；
增加列：alter table 表名 add 列名 数据类型；
改列名或者改列数据类型：alter table 表名 change 原列名 新/旧列名 新/旧数据类型；
